# -hackathon

个人心得:
    一个月过得真快，转眼间，编程马拉松就要落下帷幕了。但是我对编程的学习远远还没有到达终点，作为计算机专业的学生，编程是一门功课，也是我们毕业后要拿的出手的技能。
    当时参加这个马拉松比赛，是觉得自己作为计算机专业的学生，有这样一个从0到1的学习平台，要好好把握，所以才报名了这个比赛。但其实，虽说是这个专业的学生，有一定的优势，实际上，我在学校的课程还没有涉猎到前端的知识。通过这次马拉松比赛，我很高兴能提前接触到这样的知识，让我倍感充实。
    我一直以为学习编程是一个枯燥的东西，只能靠自己去攻克。而马拉松编程比赛采取视频讲解加线下培训这样的模式，以及结合马拉松的做法来让我去学习编程，我觉得这样特别好，因为这样不仅可以让你不急于学习任务的上交，还能真正的学习到相关知识。我是因为最近啦啦操比赛要训练，还有校运会的举行占据了我大部分时间，但闲置下来时，我能投入到这个比赛中。我觉得这是这个比赛的最大的亮点了，就是不管你做到哪里，只要你坚持，终点站都能等着你去。就是这样，我才没有放弃的念头，因为这个比赛不计较你的考核站点任务，它在乎的是你真正能学到东西。如果还有下一次的马拉松比赛，我还是会去参加的，因为我想学到更多我想要的知识。
    其实，我最想跟赛事组工作人员说，你们辛苦了！每次看到他们在群里耐心解答同学们的疑惑，我觉得他们真好。但其实，我自己一次都没有在群里提问过，倒是私下找过那个披着code马甲的微信号工作人员，不过他可能有很多学生同时在找，所以回复有点慢。但最后他还是很耐心地问我哪里不懂，直到我自己解决了问题跟他说。
    经过一个月的比拼，我想对自己说，你真棒！同时兼顾那么多事情的你，还是走到了赛事的终点，虽然是事多推着你走下来的，但是你还是很棒!
